# Demo 视频录制指南

目标时长：3 分 30 秒左右。分 7 段录制，可 NG 重录单段，后期拼接。

---

## 一、录制前准备（15 分钟）

### 1.1 确认后端/前端已就绪

终端 A（项目根目录）：
```bash
uv sync
uv run python -m uvicorn app.main:app --port 8000 --reload --app-dir backend
```

终端 B（`frontend/` 目录）：
```bash
bun install
bun run dev
```

验证：
```bash
curl -s 'http://localhost:8000/api/index/dictionary?limit=3' | head -c 200
```
有返回 JSON 即 OK。浏览器打开 `http://localhost:5173` 看到首页。

### 1.2 预热索引

第一次访问任意检索页会触发索引构建（～5 秒）。**先手动搜一次**（任意查询）让缓存 `backend/app/data/index.pkl` 就位，正式录时就不会等。

### 1.3 清场

- 浏览器：F11 全屏，Ctrl+= 放大到 125%
- 关闭通知、即时通讯弹窗
- 清空浏览器地址栏、关掉多余标签
- 录制工具：OBS Studio 或 `wf-recorder`，分辨率固定 1920×1080 或 2560×1440
- 关掉 Claude Code、IDE 等占 CPU 的东西

### 1.4 快速复制查询串

录制时别现场打字，每一段录之前复制对应查询串：
```bash
echo -n "aerodynamics AND wing NOT flutter" | wl-copy
```

---

## 二、分镜与旁白

每段录完后留 1-2 秒静止时间再切。旁白只作参考，照自己习惯说。

---

### 镜头 1 · 标题/首页（0:00–0:10）

**画面**：浏览器 `http://localhost:5173`，首页（默认跳布尔检索）。

**操作**：无（静止 5-8 秒）。

**旁白**：
> 这是基于 Cranfield 数据集的信息检索系统，1400 篇航空动力学论文摘要。系统用 Python FastAPI + Vue 3 实现，支持四种检索模式加索引浏览。

---

### 镜头 2 · 布尔检索（0:10–0:50）

**查询**：
```
aerodynamics AND wing NOT flutter
```

**操作**：
1. 导航到"布尔检索"
2. 粘贴查询 → 回车
3. 滚动看结果 → 展开第 1 条的"查看详情"
4. 收起详情

**旁白**：
> 布尔检索用递归下降解析器处理 AND、OR、NOT 和括号嵌套。这个查询返回所有提到空气动力学和机翼、但不涉及颤振的文档。高亮显示命中的词项。虽然布尔是精确匹配，我们仍然用 TF-IDF 余弦相似度对结果排序。

---

### 镜头 3 · 短语查询（0:50–1:25）

**查询**：
```
boundary layer
```

**操作**：
1. 切"短语查询"
2. 粘贴 → 回车
3. 展开一条结果看高亮位置

**旁白**：
> 短语查询要求两个词在文档里连续出现。底层靠位置倒排表：取每个词项的 posting list 求交，再校验位置差是否为 1。预处理阶段会保留原始位置信息，即使中间有停用词也不会干扰。

---

### 镜头 4 · WordNet 扩展（1:25–2:05）

**查询**：
```
heat transfer
```

**操作**：
1. 切"查询扩展"
2. 粘贴
3. 把 max_synonyms 调到 3
4. 回车
5. **重点指出扩展词列表**（热相关的 thermal、warmth 等）

**旁白**：
> 查询扩展基于 WordNet 词汇数据库，对每个查询词取同义词扩充检索词集，弥合词汇鸿沟。这里 heat 被扩到 warmth、hotness 等，transfer 被扩到 move、shift。最终用扩展后的词集做 TF-IDF 检索。

---

### 镜头 5 · Soundex 发音矫正（2:05–2:45）

**查询**（故意拼错）：
```
bounderi flo
```

**操作**：
1. 切"发音矫正"
2. 粘贴 → 回车
3. 在结果顶部展示 **编码映射表**：
   - `bounderi → B536 → [boundari]`
   - `flo → F400 → [fail, fall, flow, full, ...]`

**旁白**：
> Soundex 把拼写不同但发音相近的词映射到同一个 4 字符编码。bounderi 被编码为 B536，命中词典里真实存在的 boundari。这对用户拼错的场景特别有用，和 WordNet 同义词扩展互补——一个处理拼写噪声，一个处理词汇多样性。

---

### 镜头 6 · 词典浏览（2:45–3:15）

**操作**：
1. 切"词典浏览"
2. 前缀输入 `aero` → 回车
3. 点击词项 `aerodynam`
4. 展开看**倒排记录表**（doc_id / tf / positions）

**旁白**：
> 词典浏览暴露了索引内部结构。每个词项能看到它出现在哪些文档、每篇文档里的频次和具体位置。这不仅是调试工具，也让系统对用户透明——可以直接看到为什么某篇文档会命中某个查询。

---

### 镜头 7 · 收尾（3:15–3:30）

**画面**：切回首页 / 缓慢摇镜扫导航栏。

**旁白**：
> 以上就是系统的四种检索模式和索引浏览功能。所有结果都基于 TF-IDF 向量空间模型的余弦相似度排序，带位置信息的倒排索引保证短语查询和拼写矫正可以落地。谢谢观看。

---

## 三、后期（可选）

- **剪辑**：kdenlive 或 shotcut，剪 NG 段，段间加 0.5 秒淡入淡出
- **字幕**：如果要交视频给老师，加一个简洁的段落标题（如"布尔检索"）浮在画面左上角
- **音量均衡**：后期用 Audacity 的 Compressor 统一响度
- **压缩**：导出 H.264 + 1080p30 + 目标 bitrate 4-6 Mbps（~100 MB 以内）

## 四、故障排查

| 症状 | 原因 | 解决 |
|------|------|------|
| 访问 `http://localhost:8000/` 404 | 后端是纯 API，没首页 | 只访问 `http://localhost:5173` 前端 |
| 第一次搜索卡 5-10 秒 | 索引正在构建 | 先随便搜一次预热 |
| NLTK `SSRF Violation` 警告 | NLTK 检查数据源的误报 | 无害，忽略 |
| Soundex 结果为空 | 索引未含用户拼错的候选词 | 换一个在 Cranfield 里常见的词根，如 `heet`（heat）|
| 前端 `/api/*` 返回 CORS 错误 | 后端没启动或端口不是 8000 | 确认后端在 8000 |
